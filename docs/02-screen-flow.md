# 화면 및 사용자 흐름

## 1. 랜딩 페이지

랜딩 페이지는 서비스의 핵심 메시지를 빠르게 전달하는 데 초점을 둔다.

- 엑셀 또는 직접 입력으로 CCD 패키지를 만든다는 서비스 메시지 제시
- 입력 방식, LLM 분석, 산출물 구조를 카드로 요약
- 워크스페이스 페이지로 바로 진입할 수 있는 CTA 제공

## 2. 워크스페이스 페이지

워크스페이스는 4단계 흐름으로 구성했다.

### Step 1. 입력 방식 선택

- 파일 업로드 중심
- 하이브리드 방식
- 직접 입력 중심

이 단계는 사용자와 입력 UX를 협의할 때 중요한 비교 포인트가 된다.

### Step 2. 옵션 입력

폼 기반 입력 영역에서 다음과 같은 값을 받는다.

- 프로젝트 명
- 생산 제품
- 시설 규모
- 배치 전략
- 클린룸 등급
- 목표 산출물
- 유틸리티 / 장비 / 싱글유즈 / 자동화 반영 여부
- 설계 메모

### Step 3. 시트형 입력 미리보기

방식 1 검토를 위해, 폼 입력값이 시트 형식으로 자동 반영되도록 구성했다.

- 화면상에서 값 수정 가능
- 추후 실제 Excel sheet preview로 대체 가능
- 현재는 브라우저 내 editable table 형태

### Step 4. CCD 패키지 초안 생성

현재 프로토타입은 실제 LLM 호출 대신, 입력값을 기반으로 다음 구조를 조합한다.

- 추천 산출물 목록
- 핵심 가정 조건
- 다음 구현 단계

이 출력 구조는 추후 실제 CCD 패키지 문서 생성 결과의 기본 뼈대로 사용할 수 있다.
